# Contributing

Thanks for considering a contribution.

## Before you start

- Search existing issues before opening a new one.
- For significant visual, interaction, or architecture changes, open an issue first to align on direction.
- Keep changes focused. Do not bundle unrelated refactors into one pull request.

## Local development

```bash
npm install
npm run dev
```

Before opening a pull request, run:

```bash
npm run build
npm run lint
```

## Pull request guidelines

- Explain what changed and why.
- Include screenshots for visual changes.
- Preserve keyboard access, focus states, reduced-motion support, and responsive behavior.
- Keep copy factual. Do not add fabricated results, projects, testimonials, or metrics.
- Update relevant documentation when behavior or setup changes.

By participating, you agree to follow the [Code of Conduct](./CODE_OF_CONDUCT.md).
